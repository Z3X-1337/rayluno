# Rayluno final judge video script

Target duration: **2:35–2:50**. Record the real Windows application. Do not simulate successful effects.

## 0:00–0:18 — The problem and thesis

**Voice-over**

> Personal assistants are usually either harmless chatbots that cannot act, or powerful agents with broad system access. Rayluno takes a third path: a local-first personal assistant that can act, but must prove what it was allowed to do.

Show the Rayluno desktop, the Local/Private indicator, and the Verified control.

## 0:18–0:40 — Natural local voice input

Press the microphone and say:

> شغّل يويتوب

Show the locally transcribed text and YouTube opening. Mention that the intentionally imperfect pronunciation/spelling is normalized locally and still resolves only to an already registered destination.

**Voice-over**

> Voice input is processed locally. Rayluno tolerates common Arabic transcription errors, but fuzzy recovery can only select an application or site already present in the allow-list.

## 0:40–1:12 — Explicit approval before a sensitive effect

Type or say:

> جهز عرض الحكام

Pause on the approval dialog. Zoom enough to show:

- selected Skill IDs;
- permissions;
- risk level;
- argument-key fingerprint;
- expiry countdown;
- Approve and Reject controls.

**Voice-over**

> The model or demo planner may propose a plan, but it cannot execute it. Deterministic code resolves registered skills and creates an expiring, plan-specific, single-use approval handle.

Click **Approve and execute**. Show Notepad and the repository opening.

> Before the first operating-system effect, Rayluno persists an execution-authorized receipt. The outcome is recorded separately.

## 1:12–1:37 — Fail closed

Return to Rayluno and enter:

> اختبر رفض مهارة غير مسجلة

Keep the blocked result visible.

**Voice-over**

> When an action does not resolve to a registered skill, Rayluno fails closed before impact. There is no general shell, eval, or unrestricted command authority behind this interface.

## 1:37–2:02 — Verifiable evidence

Open **Verified** and the Runtime Trust Center. Show:

- Authorization before effect;
- Authenticated checkpoint;
- Keyed fingerprints;
- Explicit-consent memory;
- No general command authority;
- Telemetry off;
- CHAIN VERIFIED.

**Voice-over**

> The receipt journal is hash-linked, and its head and count are protected by an installation-scoped HMAC checkpoint. Before the next action, Rayluno reloads and verifies the chain. Editing, reordering, truncation, rollback, or checkpoint tampering pauses execution.

## 2:02–2:28 — Codex and GPT-5.6

Show the repository, pull-request history, tests, and architecture documentation.

**Voice-over**

> Codex powered by GPT-5.6 was a core engineering collaborator, not a cosmetic API call. It audited the imported baseline, decomposed the system into reviewable pull requests, designed the trust boundaries, implemented bilingual and accessibility contracts, built adversarial security tests, diagnosed CI artifacts, and converted hidden guarantees into this runtime-backed Trust Center. GPT-5.6 is never granted direct operating-system control.

## 2:28–2:45 — Closing impact

Show the voice button, Today panel, and Verified indicator together.

**Voice-over**

> Rayluno is designed for people who need hands-free computing without surrendering control: blind users, people with hand injuries, creators, and busy professionals. The assistant can propose. The user authorizes. The system proves.

End card:

> **Rayluno — Personal AI that proves what it is allowed to do.**
