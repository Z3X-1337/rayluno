# سجل جاهزية إصدار Rayluno 1.0.0

هذا السجل يصف الحالة الفعلية لمرشح إصدار **Rayluno Assistant 1.0.0** على Windows x64
حتى 18 يوليو 2026. المخرجات الحالية صالحة للاختبار الداخلي، لكنها ليست إطلاقًا تجاريًا
عامًا بعد؛ ما زالت هناك بوابات توقيع ومتجر ودفع واختبار موضحة أدناه.

## الخلاصة التنفيذية

| المكوّن | الحالة |
|---|---|
| حزمة Windows الكاملة | مبنية ومتحقق منها داخليًا |
| أرشيف ZIP | مبني، وبصمته تطابق ملف SHA-256 المرافق |
| مثبّت NSIS | مكتمل ومثبت في manifest، لكنه غير موقّع بـAuthenticode |
| MSIX | مكتمل كحزمة تطوير وفحص غير موقعة؛ غير قابل للرفع إلى Microsoft Store |
| موقع الإطلاق | الإصدار 4 منشور بصورة خاصة للمالك فقط |
| الدفع والتفعيل الإنتاجيان | غير مفعّلين؛ إصدار أسرار البيئة ما زال صفرًا |
| البيع العام | غير جاهز بعد |

الملف الذي تصفه الأخرى.

## المخرجات الإرثية المحفوظة

لم تُحذف مخرجات Future Assistant القديمة ولم تُستبدل. نُقلت إلى `dist/legacy` كي
تبقى قابلة للتدقيق ومنفصلة بوضوح عن إصدار Rayluno الحالي:

- `dist/legacy/FutureAssistant-0.1.0`
- `dist/legacy/FutureAssistant-0.1.0-win-x64.zip`
- `dist/legacy/installer`
- `dist/legacy/msix`

هذه الملفات أرشيف تاريخي وليست أصول إصدار Rayluno أو ملفات صالحة للبيع الحالي.

## بوابات التحقق المنجزة

- نجحت مجموعة اختبارات Python الكاملة وفحوص Ruff في آخر تحقق للمصدر قبل بناء
  المخرجات الحالية.
- نجحت اختبارات الموقع البالغ عددها `23/23`، وفحوص typecheck وlint والبناء الإنتاجي.
- أظهر `npm audit --omit=dev` صفر ثغرات معروفة في آخر تحقق.
- طابقت بصمة ZIP المحسوبة ملف البصمة المرافق.
- تطابق manifest المثبّت مع عدد ملفات الحزمة وحجمها وبصمتي بيانات provenance.
- لم تُضمّن أسرار دفع أو مفاتيح توقيع خاصة في حزمة Windows أو أرشيف الموقع.

## المتبقي قبل الإطلاق التجاري

1. حجز الهوية في Partner Center، ثم إنشاء MSIX إنتاجي جديد بالقيم الحقيقية واجتياز
   فحوص Microsoft Store؛ لا تستخدم حزمة التطوير الحالية للرفع.
2. توقيع المثبّت المباشر بـAuthenticode، واختبار التثبيت والترقية والإزالة على جهاز
   Windows نظيف.
3. إنشاء حساب Lemon Squeezy وإكمال KYC، وإعداد منتج Founders Pro بقيمة `29 USD`
   مرة واحدة ولجهازين، ثم ضبط الأسرار واجتياز فحص commerce واختبار شراء/تفعيل/إلغاء.
4. حجز اسم التطبيق وهوية الناشر في Microsoft Store، وتجهيز المواد النهائية واجتياز
   اعتماد المتجر.
5. إجراء بحث علامة تجارية رسمي وحجز نطاق Rayluno واعتماد سياسات الدعم والاسترجاع.
6. فتح الموقع للعامة وربط التنزيل فقط بعد نجاح جميع البوابات السابقة.

إلى أن تكتمل هذه البنود، الوصف الدقيق هو: **مرشح إصدار داخلي مكتمل الحزمة والمثبّت
وحزمة MSIX التطويرية، مع MSIX المتجر الإنتاجي وبنية البيع غير مفعّلين**.
