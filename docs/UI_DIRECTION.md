# Rayluno UI Direction

## Visual principle

The desktop app should feel like a quiet command center rather than a sci-fi dashboard. Use restrained motion, high contrast, spacious typography, and strong information hierarchy.

## Primary surfaces

### Today

- greeting and current status;
- overdue count;
- due-now reminder;
- next three commitments;
- one recommended focus item;
- quick voice/text command entry.

### Skills

- skill name;
- required permission;
- risk level;
- last execution status;
- enable/disable control;
- confirmation policy.

### Memory

- facts explicitly saved by the user;
- source and saved date;
- forget control;
- no invisible profiling.

### Activity

- human-readable execution receipts;
- no raw secrets or task contents in diagnostics;
- filters for completed, blocked, and failed actions.

## Interaction states

Listening → Understanding → Planning → Confirmation → Executing → Completed/Blocked.

Each state must be visually distinct and explain what Rayluno is doing. Avoid fake terminal animations or decorative telemetry that is not backed by real runtime data.
