# الأمان والخصوصية في Rayluno Assistant / Security and privacy

هذه الوثيقة تصف سلوك تطبيق Windows 1.0.0 الحالي، ثم تميّز بوضوح الضوابط المطلوبة
للمهارات المستقبلية. لا تعني عبارة «محلي» أن التطبيق لا يستخدم الشبكة أبدًا؛ بل تعني
أن الاستيقاظ والتعرّف على الكلام والنطق وتوجيه الأوامر الأساسية لا تحتاج خادمًا.

This document describes the current 1.0.0 desktop behavior. “Local-first” means wake
detection, speech recognition, speech output, and basic command routing can run without
a cloud AI service; web, activation, update, and optional API actions still use the
network when explicitly configured or requested.

## ما يبقى محليًا

- يلتقط الميكروفون PCM في الذاكرة، ويستخدمه Vosk/Whisper، ولا يكتب تسجيلات صوتية
  إلى القرص.
- يعمل اكتشاف عبارتي الاستيقاظ العربية والإنجليزية محليًا.
- يستدعي النطق أصوات Windows المثبتة محليًا؛ لا يرفع النص إلى خدمة نطق من المشروع.
- تمر الأوامر الشائعة أولًا عبر موجّه حتمي سريع، ويمكن للمستخدم اختيار Ollama المحلي
  للأوامر المركبة.
- لا توجد تحليلات استخدام أو تقارير أعطال مرسلة افتراضيًا. وجود خيار
  `telemetry_opt_in` في ملف الإعدادات لا يفعّل إرسالًا غير موجود أو خفيًا.

## متى تستخدم الشبكة؟

| الحالة | البيانات الخارجة | الضابط الحالي |
|---|---|---|
| فتح/بحث Google أو YouTube | عنوان الصفحة أو استعلام البحث عبر المتصفح | يحدث لأن المستخدم طلب خدمة ويب؛ تطبق شروط وخصوصية الخدمة |
| YouTube Data API الاختياري | استعلام الفيديو ومفتاح API الذي وفره المستخدم | المفتاح في ترويسة إلى أصل Google API ثابت، مع رفض redirects؛ يعود التطبيق لصفحة البحث عند الفشل |
| Ollama | نص الطلب إلى `127.0.0.1` افتراضيًا | اختيار المستخدم؛ لا يُثبت Ollama أو نموذج تلقائيًا |
| شراء Founders Pro | بيانات الدفع التي يدخلها العميل في checkout | يعالجها Lemon Squeezy بصفته Merchant of Record بعد تفعيل الحساب؛ تطبيق Windows لا يرى رقم البطاقة. checkout ليس live بعد |
| تفعيل Pro | مفتاح شراء، UUID تثبيت عشوائي، إصدار التطبيق واللغة | HTTPS فقط، لا redirects، ولا بصمة عتاد؛ تتحقق الخدمة من ترخيص Lemon Squeezy ومفتاح الشراء لا يُخزن محليًا |
| تجديد Pro | رمز تجديد معتم وUUID التثبيت | الرمز محمي بـ Windows DPAPI ولا يظهر لواجهة الويب |
| فحص تحديث | طلب بيان موقّع ثم ملف المثبّت عند موافقة المستخدم على التنزيل | HTTPS وتحقق Ed25519 والحجم وSHA-256؛ لا تشغيل تلقائي |

إذا غيّر مشغّل التطوير عناوين الخدمات أو أضاف مزودًا سحابيًا، فعليه تحديث سياسة
الخصوصية والإفصاح داخل المنتج قبل جمع أي بيانات.

## البيانات المخزنة محليًا

- الإعدادات المسموحة فقط في مجلد بيانات المنتج، واسمه العام الجديد
  `%LOCALAPPDATA%\Rayluno\settings.json`: الاسم، اللغة، عبارتا الاستيقاظ،
  إعداد STT/Ollama/TTS وخيار telemetry. لا يقبل الملف حقولًا إضافية.
- رمز ترخيص Ed25519 الموقّع وحالة زمن موثوق محلية. رمز الترخيص ليس مفتاح توقيع ولا
  يمنح القدرة على إصدار تراخيص جديدة.
- UUID عشوائي خاص بهذا التثبيت، وليس مشتقًا من القرص أو المعالج أو MAC address.
- حالة تجديد التفعيل المعتمة مشفرة لحساب Windows الحالي عبر DPAPI.
- سجل تدقيق JSONL اختياري. يخزن SHA-256 لنص الأمر، نوع الفعل، والنطاق/مسار URL
  ومفاتيح query دون قيمها. لا يخزن الأمر الخام أو استعلام البحث أو الصوت.

يمكن تعطيل سجل التدقيق بوضع `RAYLUNO_AUDIT_PATH` فارغًا. إزالة Pro تحذف
الترخيص وحالة التجديد؛ لا ينبغي لبرنامج إلغاء التثبيت حذف بيانات المستخدم تلقائيًا
دون خيار واضح، ولذلك يحتفظ المثبّت الحالي بمجلد بيانات المستخدم عند الإزالة.

تتعرف الترقية على `%LOCALAPPDATA%\FutureAssistant` بوصفه alias تقنيًا للبيتا
السابقة، وتهاجره أو تستخدمه بأمان كي لا تفقد بيانات المستخدم. كما تبقى متغيرات
`FUTURE_ASSISTANT_*` ومعرّفات `future-assistant` الداخلية للتحديث والتوقيع aliases
توافقية ثابتة؛ أمثلة الإعداد الجديدة تستخدم `RAYLUNO_*` وهوية المنتج المعروضة هي
Rayluno Assistant.

## الأسرار والمفاتيح

- لا يدخل مفتاح Ed25519 الخاص بالترخيص أو التحديثات في المستودع أو المثبّت. يتضمن
  العميل مفاتيح التحقق العامة فقط.
- لا تدخل مفاتيح مزود الدفع في تطبيق Windows.
- `RAYLUNO_YOUTUBE_API_KEY` مفتاح BYOK يقرأه التطبيق من بيئة العملية.
  التطبيق لا يخزنه في `settings.json`، لكن متغيرات البيئة ليست مخزن أسرار مثاليًا؛
  عند الانتقال إلى توزيع عام يجب توفير إدخال محمي عبر Windows Credential Manager
  أو DPAPI بدل مطالبة المستخدم بضبط متغير دائم.
- لا تضع أسرارًا حقيقية في `.env.example` أو Git أو سطر أوامر يمكن تسجيله.

## حدود التنفيذ الحالية

- مجموعة الأفعال الحالية محددة: فتح URL مسموح، بحث، فتح تطبيق معروف، التحكم بالصوت،
  والوقت. أسماء التطبيقات تتحول إلى معرفات ثابتة وليست أوامر shell.
- يفحص التطبيق scheme والنطاق وطول URL والاستعلام قبل التنفيذ.
- النموذج اللغوي يرى مفردات أدوات محددة ولا يحصل على PowerShell خام أو shell أو وصول
  عام لنظام الملفات.
- محتوى صفحة ويب لا يستطيع منح نفسه صلاحيات جديدة؛ النسخة الحالية لا تستخدم محتوى
  الصفحة لتوليد فعل حساس.

## سياسة المهارات المستقبلية

| الفئة | أمثلة | السياسة المطلوبة قبل إضافتها |
|---|---|---|
| آمنة | فتح تطبيق معروف، بحث، الوقت، رفع الصوت | تنفيذ مباشر مع إشعار |
| قابلة للتراجع | تعديل ملف، نقل عنصر، إضافة موعد | معاينة وتراجع واضحان |
| حساسة | إرسال رسالة، رفع ملف، مشاركة شاشة | تأكيد مرئي يذكر الجهة والمحتوى |
| مالية/تدميرية | شراء، حذف دائم، تثبيت، صلاحية مدير | تأكيد صريح ومصادقة نظام عند الحاجة |
| محظورة | استخراج كلمات مرور، تعطيل الحماية، shell يولّده النموذج | رفض |

تُعامل صفحات الويب والملفات والرسائل كبيانات غير موثوقة لمقاومة اختطاف الوكيل:
لا تنتقل تعليماتها إلى طبقة الصلاحيات، ولا تستدعي كتابة أو إرسالًا بسبب محتوى خارجي
وحده. راجع إرشادات NIST حول
[اختطاف الوكلاء](https://www.nist.gov/news-events/news/2025/01/technical-blog-strengthening-ai-agent-hijacking-evaluations).

## الخصوصية في الأردن

قانون حماية البيانات الشخصية الأردني رقم 24 لسنة 2023 نافذ منذ 17 مارس 2024. قبل
الإطلاق العام يجب تحديد الأدوار القانونية ونطاق المعالجة، وإتمام ما ينطبق من:

- الأساس القانوني والموافقة الواضحة عندما تلزم.
- تقليل البيانات والغرض ومدة الاحتفاظ وحقوق الوصول والتصحيح والحذف والاعتراض.
- إجراءات الأمن والاستجابة للخروقات.
- مراجعة نقل البيانات خارج الأردن عند تفعيل خدمات أجنبية.
- عقود المعالجين وسجل المعالجة ومعلومات التواصل الرسمية للخصوصية.
