# المعمارية التقنية لـ Rayluno Assistant

## المبدأ

الأوامر الشائعة لا تحتاج نموذجًا لغويًا. تمر أولًا عبر موجّه حتمي سريع؛ ولا يُستدعى الذكاء المحلي إلا عندما يكون الطلب مركبًا أو حواريًا.

```text
الميكروفون
  ↓
اكتشاف الكلام + كلمة استيقاظ محلية
  ↓
تحويل الكلام إلى نص محلي
  ↓
موجّه أوامر حتمي ── إن لم يفهم ── نموذج لغة محلي
  ↓                              ↓
أداة محددة + معاملات موثقة ← قرار أداة من قائمة مسموحة
  ↓
سياسة صلاحيات / تأكيد عند الحاجة
  ↓
تنفيذ + نتيجة + نطق محلي
```

هذه البنية أسرع من إرسال كل أمر إلى نموذج، وتقلل الهلوسة ومخاطر تنفيذ أوامر غير مقصودة.

## الحزمة المختارة

### أثناء التطوير

- Python 3.11+ و`asyncio`/threads لعزل الاستماع عن الواجهة.
- `sounddevice` لالتقاط صوت PCM محليًا.
- Vosk أو نموذج wake word مخصص للاستيقاظ.
- `pywhispercpp`/`whisper.cpp` هو مسار STT الافتراضي المتعدد اللغات، ويعمل من الذاكرة
  على معالجات x64 أقدم. `faster-whisper` خيار تطوير للأجهزة الحديثة فقط.
- Ollama للوصول السهل إلى نموذج لغة محلي ودعم tool calling.
- pywebview لواجهة Windows أصلية خفيفة مبنية بـHTML/CSS بلا Electron.
- SQLite للإعدادات وسجل الأفعال، مع عدم حفظ النصوص افتراضيًا.

### حزمة Windows الصوتية الحالية 1.0.0

- حزمة PyInstaller x64 تضم `whisper.cpp` وWhisper base ونموذجي Vosk العربي والإنجليزي.
- واجهة pywebview وCLI منفصلان داخل `dist/Rayluno`، ويوجد أرشيف ZIP للإصدار 1.0.0.
- Windows TTS يستدعي أصوات النظام الموجودة ولا يعيد توزيعها.
- تراخيص Ed25519 وتحديثات Ed25519 مضمّنة بمفاتيح عامة فقط.
- Ollama اختياري وغير مضمّن؛ لا يوجد `llama.cpp` أو نموذج LLM داخل الحزمة الحالية.
- لم يُبنَ بعد مثبّت NSIS أوMSIX جديدان لـRayluno 1.0.0. مرشحو 0.1.0 السابقون
  محفوظون تحت `dist/legacy` للتحقق التاريخي فقط، وليسوا أصول نشر للهوية الحالية.

المراجع الرسمية: [faster-whisper](https://github.com/SYSTRAN/faster-whisper)، [whisper.cpp](https://github.com/ggml-org/whisper.cpp)، [Ollama tool calling](https://docs.ollama.com/capabilities/tool-calling)، [llama.cpp](https://github.com/ggml-org/llama.cpp).

## قرار كلمة الاستيقاظ

كلمتا الاستيقاظ الافتراضيتان للهوية المختارة هما `يا رايلونو` و`Hey Rayluno`.
يعمل اكتشافهما محليًا، ويمكن للمستخدم تغييرهما من الإعدادات.

كود `openWakeWord` مرخص Apache-2.0، لكن النماذج الجاهزة المرفقة مرخصة CC BY-NC-SA 4.0 ولا تصلح لأن تُشحن داخل منتج ربحي. لذلك:

- النسخة التطويرية تستخدم Vosk مع نموذج ذي ترخيص Apache-2.0 أو وضع الضغط للتحدث.
- النسخة التجارية تستخدم اسم Rayluno الأصلي ونموذجًا ندربه من تسجيلات أو بيانات اصطناعية مرخصة بوضوح.
- لا تُحفظ عينات الميكروفون ولا تُرفع تلقائيًا لجمع بيانات تدريب.

يوجد [نموذج Vosk عربي MGB2](https://alphacephei.com/vosk/models) بأرشيف تنزيل
يقارب 318MB وترخيص Apache-2.0؛ حجمه المفكوك في حزمة 1.0.0 الحالية يقارب 698MB.
هو حل تعرف مستمر وليس كاشف استيقاظ متخصصًا، لذلك يجب قياس استهلاكه ومعدل التفعيل
الخاطئ على أجهزة بيتا متعددة قبل الإطلاق العام.

## قرار التعرف على الكلام

- الجهاز الضعيف: Whisper `base` مكمّم، مع لغة عربية افتراضية قابلة للتغيير.
- الجهاز المتوسط: Whisper `small` مكمّم؛ غالبًا أفضل توازن للعربية.
- الجهاز القوي: نموذج أكبر أو Qwen3-ASR بعد قياس حقيقي.

بيئة التطوير لا تنزّل نموذجًا كبيرًا تلقائيًا. أما حزمة Windows الصوتية الكاملة فتضم
Whisper base ونموذجي Vosk كي تعمل بلا تنزيل أولي؛ يجب عرض حجم المثبّت بوضوح للمستخدم.

## قرار نموذج اللغة

نحتاج نموذجًا متعدد اللغات يدعم استدعاء الأدوات، لكن اسم النموذج وحجمه يظلان إعدادًا لا اعتمادًا ثابتًا. طبقة المزود تفصل التطبيق عن Ollama/llama.cpp أو API يختاره المستخدم.

النموذج لا يحصل على PowerShell خام. يرى فقط أدوات مثل:

```text
open_url(url)
search_web(query)
search_youtube(query)
launch_app(app_id)
change_volume(direction, steps)
get_local_time()
```

يتحقق التطبيق من اسم الأداة ومعاملاتها وسياسة صلاحيتها بعد خروجها من النموذج وقبل التنفيذ.

## مستويات الأجهزة المستهدفة

| المستوى | حد عملي أولي | الاختيار الافتراضي |
|---|---|---|
| Basic | 8GB RAM، CPU فقط | أوامر حتمية + Whisper base + نموذج لغة صغير اختياري |
| Standard | 16GB RAM، CPU حديث أو GPU 6GB | Whisper small + LLM محلي 4B مكمّم |
| Pro | 32GB RAM، GPU 10GB+ | STT/LLM أكبر وصوت عصبي محلي |

هذه حدود بدء للاختبار، وليست متطلبات نهائية قبل benchmark على أجهزة فعلية.

## حالة التنفيذ وخارطة ما تبقى

1. **مكتمل في 1.0.0:** موجّه عربي/إنجليزي، أفعال آمنة، سجل منزوع المحتوى، واجهة
   ثنائية اللغة، onboarding وإعدادات محلية.
2. **مكتمل للبيتا:** عبارتا استيقاظ، التقاط/تسجيل utterance في الذاكرة، STT/TTS،
   زر إيقاف الاستماع، وحزمة صوت offline كاملة.
3. **مكتمل اختياريًا:** Ollama محلي مع tool calling مضبوط، بلا shell خام.
4. **مكتمل كأساس تجاري:** Free/Pro، تفعيل purchase-key، تجديد DPAPI، تحديثات موقّعة،
   NSIS per-user، وسياسة خصوصية تقنية.
5. **غير مكتمل للإطلاق العام:** البحث القانوني الرسمي لعلامة Rayluno وحجز النطاق
   واعتماد Partner Center، توقيع Authenticode عند التوزيع المباشر، SBOM وإشعارات
   نهائية، KYC ومنتج Lemon Squeezy وتفعيلهما، قناة تحديث فعلية، دعم واسترجاع، وبيتا
   أجهزة متعددة. لا توجد حسابات دفع أو Store مفعلة في حالة 1.0.0 الحالية.
6. **خارطة لاحقة:** مهارات Windows/متصفح محددة قابلة للإلغاء مع تأكيدات، لا وصول
   عام يولّده النموذج.
